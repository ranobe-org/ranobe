## Основные правила

- Вы (участник/разработчик) можете изменять любой участок кода по своему усмотрению.
- Несколько правил
    - Константы (`Constants`) должны быть объявлены в `config/Ranobe.java`.
    - Соблюдайте требования к чистому коду.
    - Старайтесь не добавлять сторонние библиотеки (из‑за размера пакета).
    - По возможности минимизируйте размер приложения (это будет отлично).

## Добавление нового источника

- Файл для нового источника должен быть добавлен в директорию `/sources/<lang>/`.
- Имя файла должно быть `<SiteName>.java`.
- Реализуйте интерфейс `Source.java` для добавления нового источника.
- Если по каким‑то причинам реализовать метод невозможно — выбросите корректный `Exception` с сообщением.
    - Например: ```throw new Exception("Извините")```
- Перед созданием PR тщательно протестируйте новый источник.
- После завершения отправьте PR с заголовком `Add new source: <SOURCE_NAME_HERE>`; имя ветки может быть `source/<source_name>`.

## Правила

- Не определяйте поле `id` для `Novel` и `NovelItem`, так как оно вычисляется в конструкторе.
- Не определяйте поле `novelId` для `Chapter` и `ChapterItem`, так как оно также вычисляется в конструкторе;
  поле `id` должно содержать номер главы в ранобэ.
